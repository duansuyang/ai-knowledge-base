# Codevira MCP - AI 编码智能体的持久记忆

## 1. 项目概述

### 1.1 项目名称
Codevira MCP

### 1.2 一句话概述
基于 Model Context Protocol 的持久记忆和项目上下文工具，为 AI 编码智能体提供跨会话的记忆能力。

### 1.3 详细描述
Codevira 是一个 MCP 服务器，可嵌入任何项目，为处理代码库的每个 AI 智能体提供共享的持久记忆。它提供上下文图、语义代码搜索、路线图、变更集跟踪、决策日志和智能体角色定义等功能，将每会话 15,000+ tokens 的重复发现减少到约 1,400 tokens。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 上下文图 (Context Graph)
- **描述**: 每个源文件都有一个节点，包含角色、规则、依赖关系、稳定性、`do_not_revert` 标志
- **为什么重要**: 让智能体理解代码库的结构和历史
- **对比现有方法**: 传统方式每次会话都需要重新分析代码库

#### 创新点 2: 语义代码搜索
- **描述**: 自然语言搜索整个代码库，无需 grep 或读取文件
- **为什么重要**: 提高智能体定位代码的效率
- **创新性**: 将语义搜索引入代码智能体

#### 创新点 3: 路线图跟踪
- **描述**: 基于阶段的跟踪器，让智能体始终知道当前处于哪个阶段
- **为什么重要**: 避免智能体做出与项目进度矛盾的决策

#### 创新点 4: 决策日志
- **描述**: 每个会话写入结构化日志，未来智能体可搜索过去的决策
- **为什么重要**: 建立项目决策的历史记录

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|---------|
| 成本降低 | 减少重复发现开销 | 15,000+ → 1,400 tokens/会话 |
| 兼容性 | 支持多种 AI 工具 | Claude Code, Cursor, Windsurf 等 |
| 标准化 | 基于 MCP 协议 | 跨平台兼容 |
| 原子性 | 多文件变更跟踪 | 支持中断后恢复 |

### 3.3 应用场景

1. **大型项目开发**: 跨多个会话维护代码库上下文
2. **团队协作**: 多个智能体共享项目状态和决策
3. **代码库理解**: 快速理解项目结构和历史
4. **智能体教育**: 定义明确的智能体角色和协议

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 MCP 服务器架构
Codevira 作为 Model Context Protocol 服务器运行，提供以下核心能力：

| 能力 | 功能描述 |
|------|---------|
| 上下文图 | 代码文件的知识图谱表示 |
| 语义搜索 | 基于自然语言的代码搜索 |
| 路线图 | 项目阶段跟踪 |
| 变更集 | 原子性多文件变更跟踪 |
| 决策日志 | 结构化决策记录 |
| 智能体角色 | 7 种角色定义 |

#### 4.1.2 智能体生命周期

```
Start Session → Check Open Changesets → Get Project Roadmap → 
Search Past Decisions → Load Graph Context → Plan Task → 
Implement Code → Run Tests → Update Graph → Write Session Log
```
