# RF-Agent: Automated Reward Function Design via Language Agent Tree Search

## 1. 项目概述

### 1.1 项目名称
RF-Agent (Reward Function Agent)

### 1.2 一句话概述
使用语言Agent结合MCTS树搜索自动设计强化学习奖励函数，在IsaacGym和Bi-DexHands的17个控制任务上取得优异成绩。

### 1.3 详细描述
RF-Agent将大型语言模型视为语言Agent，将奖励函数设计框架化为序贯决策问题。通过集成蒙特卡洛树搜索(MCTS)来管理奖励设计和优化过程，有效利用LLM的多阶段上下文推理能力。该方法已接受为NeurIPS 2025 Spotlight论文。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 序贯决策视角重新定义奖励函数设计
- **描述**: 将LLM-based奖励函数设计问题从传统的贪婪/进化方法转变为序贯决策问题
- **为什么重要**: 传统方法缺乏对历史反馈的有效利用和复杂控制任务中的高效搜索
- **对比现有方法**: 相比Greedy和Evolutionary方法，能更好地利用历史信息和提高搜索效率

#### 创新点 2: MCTS集成的语言Agent框架
- **描述**: 将蒙特卡洛树搜索(MCTS)与语言Agent结合，管理奖励设计和优化过程
- **为什么重要**: 利用LLM的多阶段上下文推理能力，实现更高效的奖励函数搜索
- **对比现有方法**: 相比Eureka等方法，具有更好的搜索效率和性能

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 方法名称
Language Agent Tree Search (LATS) for Reward Function Design

#### 4.1.2 方法描述
RF-Agent将LLM视为语言Agent，将奖励函数设计问题框架化为序贯决策过程。该方法：
1. 使用LLM生成候选奖励函数
2. 利用MCTS管理搜索空间
3. 通过多阶段推理利用历史反馈
4. 在IsaacGym和Bi-DexHands环境上进行评估

### 4.2 模型结构

| 参数 | 规模 |
|------|-----|
| LLM后端 | GPT-4o-mini (推荐) / GPT-4o |
| 任务数量 | 17个低层次控制任务 |
| 搜索次数 | IsaacGym: 80, Bi-DexHands: 512 |

### 4.3 技术栈

| 类别 | 技术/工具 |
|------|----------|
| 框架 | IsaacGym, PyTorch |
| LLM | OpenAI API (GPT-4o-mini, GPT-4o) |
| 搜索算法 | MCTS |

## 6. 总结与评价

### 6.1 综合评价
RF-Agent是一个创新性的强化学习奖励函数自动设计框架，通过将LLM作为语言Agent并结合MCTS树搜索，成功实现了高效的奖励函数自动设计。该工作已被NeurIPS 2025接收为Spotlight论文，证明了其学术价值和实用价值。

### 6.2 值得关注的点

1. **创新性**: 首次将MCTS与语言Agent结合用于奖励函数设计
2. **实用性**: 可为各种机器人控制任务自动设计高效奖励函数
3. **可扩展性**: 框架可应用于其他RL任务
