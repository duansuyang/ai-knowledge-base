# bug-hunt - Adversarial Bug Hunting Skill

## 1. 项目概述

### 1.1 项目名称
bug-hunt

### 1.2 一句话概述
Claude Code的对抗性Bug发现技能，3个隔离Agent相互博弈找出真实Bug。

### 1.3 详细描述
使用3个完全隔离的AI Agent进行对抗性Bug发现：Hunter(过度报告) → Skeptic(反驳) → Referee(最终裁决)。利用LLM的"sycophancy"特性，通过对抗博弈提高Bug检测质量。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 三Agent对抗框架
- **描述**: Hunter(找bug) → Skeptic(反驳) → Referee(裁决)，完全隔离的上下文
- **为什么重要**: 消除锚定偏见，提高检测准确性
- **对比现有方法**: 传统code review只有一个agent，容易被前期发现影响

#### 创新点 2: 激励博弈设计
- **描述**: Hunter按bug严重程度得分(+1/+5/+10)，Skeptic反驳成功得2倍奖励但错误反驳扣2倍
- **为什么重要**: 激励驱动各Agent发挥最大能力

#### 创新点 3: 分支Diff模式
- **描述**: 支持 `-b branch --base dev` 扫描分支变更
- **为什么重要**: 只需扫描变更文件，效率高

## 4. 技术细节

### 4.1 核心方法

#### 三Agent工作流
```
Code → Hunter(找bug，过度报告) → [bug列表] → Skeptic(反驳，筛选) → [疑似bug] → Referee(独立裁决) → 最终报告
```

#### 评分机制
- **Hunter**: +1低危/+5中危/+10高危bug
- **Skeptic**: 反驳成功+奖励，错误反驳-2倍惩罚
- **Referee**: +1/-1对称评分
