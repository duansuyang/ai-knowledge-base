# Ghost - Intent-Based Git Commits

## 1. 项目概述

### 1.1 项目名称
Ghost

### 1.2 一句话概述
颠覆传统Git工作流：不是提交代码，而是提交提示词。AI生成代码的同时保留意图，每次提交都可复现。

### 1.3 详细描述
Ghost是一个CLI工具，它颠覆了传统的git工作流：不是提交代码，而是提交提示词（prompts）。AI编码代理生成产物（artifacts），而提交记录捕获了意图和输出。你的git历史变成了一条提示词及其结果的链条。

## 3. 项目优势

### 3.1 核心创新点

**意图提交（Intent-Based Commits）**: 代码是临时的，意图是永久的。每个`ghost commit`回答的是"我想要发生什么？"，而不是"什么字节改变了"。

**可复现性**: 每个提交都保留了提示词以及使用的模型和代理信息，可以随时重新运行来查看AI从相同指令生成了什么。

**设计文档化**: git log变成了设计文档，从上往下读可以看到每个架构决策背后的意图。

### 3.2 技术优势

| 优势类别 | 具体优势 |
|---------|---------|
| 可追溯性 | 完整的意图记录 |
| 可复现性 | 任何提交都可重新运行 |
| 上下文完整 | diff显示AI的决定，消息显示你要求的 |

### 3.3 应用场景

1. **AI辅助开发团队**: 记录AI生成的每个代码变更的意图
2. **代码审查**: 通过意图理解AI生成的代码
3. **知识传承**: 新成员可以通过git log理解整个开发过程
