# Ghost - Git is the Coding Agent

## 1. 项目概述

### 1.1 项目名称
Ghost

### 1.2 一句话概述
一个将 Git 工作流与 AI 编码代理深度结合的工具，让 Git 本身成为编程代理。

### 1.3 详细描述
Ghost 是一个创新的编码代理框架，它将 Git 版本控制系统的核心概念与 AI 代理能力相结合。该项目探索了"代码即代理"的理念，通过 Git 的分支、提交、合并等概念来驱动 AI 编码任务。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: Git 原生代理架构
- **描述**: 将 AI 代理的思考过程映射到 Git 的分支、提交、PR 工作流中
- **为什么重要**: 提供了一种更结构化、可追溯的 AI 编码方式
- **对比现有方法**: 传统代理是黑盒，Ghost 将其变为可版本控制的透明过程

### 3.2 技术优势

| 优势类别 | 具体优势 |
|---------|---------|
| 可追溯性 | 所有代理决策都有 Git 提交记录 |
| 可回滚 | 可以轻松回滚到任意代理决策点 |
| 协作性 | 支持多人协作的代理工作流 |

### 3.3 应用场景

1. **复杂代码重构**: AI 代理在分支上进行重构，可审查可回滚
2. **代码审查自动化**: 利用 Git PR 流程进行 AI 辅助审查
3. **增量开发**: 大型功能由多个小提交组成，可追踪每一步

## 4. 技术细节

### 4.1 核心方法

#### 4.1.1 方法描述
Ghost 将每个 AI 代理任务建模为一个 Git 分支，通过提交（commit）来记录代理的思考过程和代码变更，最终通过 Pull Request 进行审查和合并。

### 4.2 技术栈

| 类别 | 技术/工具 |
|------|----------|
| 核心框架 | Shell, Git |
| AI 集成 | 兼容多种 LLM API |
