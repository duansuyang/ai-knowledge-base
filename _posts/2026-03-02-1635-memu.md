# memU - 24/7主动式AI记忆框架

## 1. 项目概述

### 1.1 项目名称
memU

### 1.2 一句话概述
为24/7持续运行的主动式AI智能体设计的记忆框架，大幅降低保持智能体在线的LLM Token成本。

### 1.3 详细描述
memU是一个专门为24/7主动式智能体设计的记忆框架。它能够持续捕获和理解用户意图，即使没有明确指令，Agent也能主动预测用户即将要做的事情并提前行动。memU将记忆组织为类似文件系统的层级结构，支持自动分类、跨引用和上下文预测。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 主动式记忆
- **描述**: Agent无需等待用户指令，主动预测用户意图并提前准备上下文
- **为什么重要**: 实现真正的"主动式AI助手"，而不只是被动响应
- **对比现有方法**: 传统RAG需要用户查询触发，memU主动监控和学习

#### 创新点 2: 文件系统式记忆架构
- **描述**: 将记忆组织为层级结构（Categories → Items → Resources），类似文件系统
- **为什么重要**: 便于导航、搜索和维护记忆
- **对比现有方法**: 比平面向量存储更结构化

#### 创新点 3: 成本优化
- **描述**: 通过记忆缓存减少冗余LLM调用，Token成本降低至1/10
- **为什么重要**: 使24/7在线Agent在经济上可行
- **对比现有方法**: 比持续传递完整上下文更经济

## 4. 技术细节

### 4.1 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                     USER QUERY                              │
└─────────────────────────────────────────────────────────────┘
                              │
        ┌─────────────────────┴─────────────────────┐
        ▼                                           ▼
┌───────────────────┐                     ┌───────────────────┐
│   🤖 MAIN AGENT  │                     │   🧠 MEMU BOT     │
│   执行任务/响应   │◄──────────────────►│  监控/记忆/主动  │
└───────────────────┘                     └───────────────────┘
        │                                           │
        │  Memory Layer:                            │
        │  - Category (类别摘要)                    │
        │  - Item (具体记忆)                        │
        │  - Resource (原始资源)                    │
```

### 4.2 核心API

```python
# 持续学习
result = await service.memorize(
    resource_url="path/to/file.json",
    modality="conversation",  # or document/image/video/audio
    user={"user_id": "123"}
)

# 主动检索 (RAG模式 - 快速)
result = await service.retrieve(
    queries=[...],
    method="rag"  # 毫秒级响应
)

# 主动检索 (LLM模式 - 深度推理)
result = await service.retrieve(
    queries=[...],
    method="llm"  # 意图预测，更深度
)
```
