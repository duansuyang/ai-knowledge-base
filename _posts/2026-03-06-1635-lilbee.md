# lilbee - Local RAG CLI and MCP Server

## 1. 项目概述

### 1.1 项目名称
lilbee

### 1.2 一句话概述
本地RAG CLI和MCP服务器，索引文档和代码，通过终端或Agent工具提问。

### 1.3 详细描述
完全本地运行的RAG工具，基于Ollama + LanceDB，无需Docker、外部数据库或云API。人机交互用交互式聊天，Agent通过MCP服务器或JSON CLI直接搜索索引文档。

### 1.4 所属领域
- [x] RAG/知识库 (向量检索、Hybrid Search)

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 完全本地化
- **描述**: 基于Ollama + LanceDB，无云服务依赖
- **为什么重要**: 数据隐私优先用户的需求
- **对比现有方法**: 大多数RAG方案需要云API或Docker

#### 创新点 2: MCP服务器
- **描述**: 暴露MCP服务器，Agent可直接调用
- **为什么重要**: 让Agent能自主搜索知识库，而非依赖LLM生成答案

#### 创新点 3: 分离搜索和问答
- **描述**: `search`返回嵌入的chunks(不需要Ollama)，`ask`调用Ollama生成答案
- **为什么重要**: Agent可以用自己的LLM推理搜索结果，更灵活

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|---------|
| 隐私 | 完全本地运行 | 无云API |
| 灵活性 | search/ask分离 | Agent自选LLM |
| 易用性 | CLI + MCP双接口 | 即装即用 |

### 3.3 应用场景

1. **本地知识库**: 隐私敏感的文档问答
2. **开发辅助**: 代码库索引，Agent问答
3. **离线使用**: 无网络环境下的RAG

## 4. 技术细节

### 4.1 核心方法

#### 工作流程
```
文档 → sync(提取文本+chunk+embedding) → LanceDB存储
                                            ↓
Query → search(向量检索) → 返回chunks → ask(Ollama生成答案) → 带引用答案
```

#### 双接口
- **人类**: 交互式聊天 (streaming响应, tab补全, slash命令)
- **Agent**: MCP服务器 + JSON CLI

### 4.2 技术栈

| 类别 | 技术/工具 |
|------|----------|
| 运行时 | Python 3.11+ |
| LLM/Embedding | Ollama (mistral, nomic-embed-text) |
| 向量数据库 | LanceDB |
| OCR | Tesseract (可选) |

## 5. 总结与评价

### 5.1 综合评价

Lilbee是一个定位清晰的本地RAG工具，2 stars因为是新项目。核心亮点是完全本地化 + MCP支持 + search/ask分离设计。对于需要在本地或私有环境部署RAG的用户，这个项目值得关注。唯一需要注意的是项目标注为"early experimental"。
