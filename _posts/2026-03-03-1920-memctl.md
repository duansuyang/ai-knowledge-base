# memctl - Persistent Memory for AI Agents

## 1. 项目概述

### 1.1 一句话概述
用 SQLite 实现 AI 智能体的持久化内存，无需基础设施，一个文件即可实现语义搜索和记忆衰减。

### 1.3 详细描述
memctl 为 AI 智能体提供持久化记忆能力，通过单个 SQLite 文件存储，支持嵌入式向量搜索和可配置的内存衰减机制。解决智能体跨会话遗忘问题，对标 Mem0 和 LangMem 但无需额外基础设施。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 零基础设施
- **描述**: 仅需一个 SQLite 文件 (`~/.memctl/memory.db`)，无需 Docker、Postgres 或向量数据库
- **为什么重要**: 大幅降低 AI 智能体记忆系统的部署门槛和成本
- **对比现有方法**: Mem0 需要 Docker + Postgres + Qdrant，定价从 $19 到 $249

#### 创新点 2: 嵌入式向量搜索
- **描述**: 内置向量嵌入功能，无需调用外部 API
- **为什么重要**: 降低延迟和成本，保护数据隐私
- **对比现有方法**: 传统方案依赖外部向量数据库服务

#### 创新点 3: 智能记忆衰减
- **描述**: 可配置的遗忘曲线（指数、线性、阶梯）
- **为什么重要**: 让智能体记住重要信息，遗忘无关信息，避免记忆膨胀
- **对比现有方法**: 文本文件方案无衰减机制，记忆无限累积

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |

## 4. 技术细节

### 4.1 核心方法

#### 4.1.1 存储架构
- 单 SQLite 文件存储 (`~/.memctl/memory.db`)
- 内置向量嵌入（嵌入式模型）
- JSON 元数据存储（标签、智能体命名空间）

#### 4.1.2 CLI 命令

```bash
# 存储事实
memctl store "Polar API expects price in cents" --agent myagent --tags api,billing

# 语义搜索召回
memctl recall "Polar pricing" --agent myagent

# 列出近期记忆
memctl list --agent myagent --since 7d

# 记忆衰减（遗忘不常用的记忆）
memctl decay --dry-run
memctl decay --apply

# 查看统计
memctl stats
```

#### 4.1.3 MCP 服务器

```bash
memctl mcp serve
```

支持 Claude Desktop、Cursor、Windsurf 等编辑器。

### 5.2 局限性分析

#### 5.2.1 已知的局限性

1. **向量搜索精度**: 嵌入式向量搜索精度可能不如专用向量数据库

### 7.2 综合评价

memctl 是一个解决 AI 智能体记忆问题的实用工具。其核心优势是零基础设施——仅需一个 SQLite 文件即可实现持久化记忆、语义搜索和智能衰减。相比动辄需要 Docker + Postgres + Qdrant 的企业方案，memctl 大幅降低了门槛。

适合：个人开发者、研究者、需要轻量级记忆方案的团队。
不适合：需要超大规模向量检索的企业级场景。

### 7.3 值得关注的点

1. **轻量级**: 一个文件搞定一切
2. **MCP 集成**: 原生支持 Model Context Protocol
3. **记忆衰减**: 智能遗忘机制是独特卖点
4. **MIT 许可**: 商业友好
