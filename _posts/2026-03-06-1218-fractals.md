# Fractals - 递归智能体任务编排器

## 1. 项目概述

### 1.1 项目名称
Fractals

### 1.2 一句话概述
递归任务编排器，将任意高级任务分解为自相似的可执行子树，并通过智能体集群在隔离的 git worktree 中运行每个叶子任务。

### 1.3 详细描述
Fractals 是一个实验性的递归智能体任务编排系统。用户输入高级任务后，系统会自动将其分解为树状结构的子任务，然后在独立的 git worktree 中通过 Claude/Codex CLI 并行执行叶子任务。支持深度优先和广度优先等多种批处理策略。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 递归任务分解
- **描述**: 将复杂任务递归分解为自相似的树状子任务结构
- **为什么重要**: 允许处理任意复杂度的任务，同时保持任务间的关联性和上下文
- **对比现有方法**: 传统的任务分解通常是扁平或单层的，Fractals 支持深度递归分解

#### 创新点 2: Git Worktree 隔离执行
- **描述**: 每个叶子任务在独立的 git worktree 中执行，避免相互干扰
- **为什么重要**: 提供了干净的隔离环境，同时保留了 git 版本控制能力
- **对比现有方法**: 比 Docker 容器更轻量，比直接在主目录执行更安全

#### 创新点 3: 智能体集群编排
- **描述**: 支持批量并行执行多个叶子任务，通过 rate limit 控制
- **为什么重要**: 充分利用计算资源，同时避免 API 限制
- **对比现有方法**: 提供了更灵活的执行策略（深度优先、广度优先、分层顺序）

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|---------|
| 可扩展性 | 递归分解任意复杂度任务 | 无限制深度 |
| 并行性 | 批量叶子任务并行执行 | 受 rate limit 约束 |
| 隔离性 | git worktree 隔离 | 干净的工作环境 |
| 可视化 | Next.js 前端展示任务树 | 实时状态更新 |

### 3.3 应用场景

1. **复杂代码项目开发**: 将大型功能需求分解为多个可独立开发的小任务
2. **自动化测试生成**: 递归分析代码结构，生成单元测试
3. **文档自动化**: 将大型文档需求分解为章节、段落分别生成
4. **研究任务自动化**: 将实验需求分解为数据准备、训练、评估等子任务

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 两阶段流程

**Phase 1: PLAN (规划)**
1. 用户输入任务描述和最大深度
2. LLM 分类任务（原子任务 vs 复合任务）
3. 递归分解复合任务为子树
4. 展示完整的任务树供用户确认

**Phase 2: EXECUTE (执行)**
1. 用户确认计划后，提供工作区路径
2. 自动初始化 git 仓库和 worktree
3. 批量执行叶子任务（Claude CLI）
4. 实时轮询更新执行状态

#### 4.1.2 技术栈

| 类别 | 技术/工具 |
|------|----------|
| 后端 | Hono (Node.js) |
| 前端 | Next.js |
| LLM | OpenAI (GPT-5.2) |
| CLI | Claude Code / Codex CLI |
| 版本控制 | Git Worktree |

### 5.2 综合评价

Fractals 是一个有趣的实验性项目，尝试解决复杂任务的自动化分解和执行问题。其核心创新在于使用递归方式将任务分解为树状结构，并通过 git worktree 实现任务隔离。适合需要处理复杂多步骤任务的开发者使用。不过目前仍在实验阶段，稳定性有待验证。

### 5.3 值得关注的点

1. **任务分解的智能化**: 利用 LLM 进行任务分类和分解
2. **执行策略的多样性**: 支持深度优先、广度优先等多种策略
3. **与现有工具的集成**: 深度集成 Claude CLI 和 Git
