# curgit - 高性能Git Agent CLI工具

## 1. 项目概述

### 1.1 项目名称
curgit

### 1.2 一句话概述
用Rust编写的高性能CLI工具，作为独立的Git Agent，自动分析staged changes并生成符合Conventional Commits标准的专业commit message。

### 1.3 详细描述
curgit是一个用Rust编写的高性能CLI工具，充当独立的Git Agent。它能分析git仓库中的staged changes，使用LLM生成符合Conventional Commits标准的专业、上下文感知的commit message。支持多种LLM provider（Cursor、Ollama、OpenAI、Claude、Kimi、DeepSeek等）。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 高性能Rust实现
- **描述**: 整个工具使用Rust编写，相比Python/Node.js版本有显著的性能优势
- **为什么重要**: CLI工具需要快速响应，Rust的零成本抽象确保了高效执行
- **对比现有方法**: 现有git commit辅助工具多为Python/Shell脚本，性能较差

#### 创新点 2: 智能Diff分析
- **描述**: 自动提取staged changes、文件名、hunks和函数签名
- **为什么重要**: 生成准确的commit message需要理解代码的上下文
- **对比现有方法**: 传统工具只分析文件级别变化，忽略函数级别语义

#### 创新点 3: Auto-Split自动拆分
- **描述**: 当diff过大时，自动将大型变更拆分为多个原子commit
- **为什么重要**: 保持commit的原子性是最佳实践，手动拆分费时费力
- **触发阈值**: 8+文件 / 20+hunks / 20,000+字符

## 4. 技术细节

### 4.1 核心方法

#### 4.1.1 方法描述
curgit的工作流程：
1. 检测git仓库的staged changes
2. 过滤噪音文件（lock文件、二进制文件等）
3. 将diff发送给LLM进行分析
4. 生成符合Conventional Commits格式的message
5. 展示给用户确认或编辑

## 5. 安装与使用

### 5.1 安装

```bash
# 一键安装
./install.sh

# 或指定目录
CURGIT_INSTALL_DIR=~/.local/bin ./install.sh

# 手动安装
cargo install --path .
```

### 5.2 快速开始

```bash
git add .
curgit
```

### 5.3 配置

配置文件位置: `~/.config/curgit/config.toml`
