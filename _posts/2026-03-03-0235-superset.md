# Superset - AI 编码代理终端

## 1. 项目概述

### 1.1 项目名称
Superset

### 1.2 一句话概述
AI 时代的终端应用，可并行运行多个 Claude Code、Codex、Cursor 等 AI 编码代理，实现高效的多任务开发。

### 1.3 详细描述
Superset 是一个专为 AI 编码代理设计的终端应用，支持同时运行 10+ 编码代理而无需切换上下文。每个任务在独立的 git worktree 中运行，避免相互干扰。支持 Claude Code、OpenAI Codex、Cursor、 Gemini CLI、GitHub Copilot 等主流代理。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 并行代理执行
- **描述**: 支持同时运行 10+ AI 编码代理，显著提升开发效率
- **为什么重要**: 传统方式需要串行执行多个任务，Superset 实现真正的并行处理
- **对比现有方法**: 需要打开多个终端窗口，管理复杂

#### 创新点 2: Git Worktree 隔离
- **描述**: 每个任务在独立的 git worktree 中运行
- **为什么重要**: 避免不同代理之间相互干扰，保证代码隔离
- **对比现有方法**: 多代理共享工作目录容易导致冲突

#### 创新点 3: 统一监控面板
- **描述**: 从单一界面监控所有代理状态
- **为什么重要**: 实时了解任务进度，需要时快速介入
- **对比现有方法**: 需要在多个终端之间切换查看

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|---------|
| 并行能力 | 10+ 代理同时运行 | 提升效率 10x |
| 隔离性 | Git worktree 隔离 | 零冲突 |
| 兼容性 | 支持所有 CLI 代理 | Claude/Codex/Cursor |
| 集成 | 内置 Diff 查看器 | 无缝审查 |
| 快捷键 | 完整键盘支持 | 高效导航 |

### 3.3 应用场景

1. **大规模代码重构**: 同时运行多个代理处理不同模块
2. **并行测试**: 多个代理同时运行不同测试套件
3. **文档生成**: 同时生成多个项目的文档
4. **Bug 修复**: 多代理并行调查不同问题

**优点**:
- 真正实现多代理并行
- 隔离机制安全可靠
- 键盘操作高效

**局限**:
- 目前仅支持 macOS
