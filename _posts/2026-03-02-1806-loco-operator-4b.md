# LocoOperator-4B: 本地运行的代码Agent模型

## 1. 项目概述

### 1.1 项目名称
LocoOperator-4B

### 1.2 一句话概述
4B参数的工具调用Agent模型，通过从Qwen3-Coder-Next推理轨迹进行知识蒸馏训练，专注多轮代码库探索，可本地通过llama.cpp零API成本运行。

### 1.3 详细描述
LocoOperator-4B是一个4B参数的代码Agent模型，通过全参数SFT从Qwen3-Coder-Next的推理轨迹进行知识蒸馏训练。它专门用于多轮代码库探索——读取文件、搜索代码、在Claude Code风格的Agent循环中导航项目结构。设计为本地子Agent，通过llama.cpp在Mac Studio上零API成本运行。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 工具调用Agent蒸馏
- **描述**: 从Qwen3-Coder-Next的推理轨迹进行知识蒸馏，训练出专注于工具调用的Agent模型
- **为什么重要**: 实现了小模型也能具备复杂代码库探索能力
- **对比现有方法**: 相比teacher模型Qwen3-Coder-Next，4B参数模型更轻量且可本地运行

#### 创新点 2: 100% JSON有效性
- **描述**: 每个工具调用都是有效JSON，包含所有必需参数
- **为什么重要**: 超越了teacher模型(87.6%)的JSON有效性
- **对比现有方法**: 老师模型有11个工具调用缺少必需参数

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 方法名称
Full-parameter SFT Knowledge Distillation

#### 4.1.2 方法描述
LocoOperator-4B使用全参数SFT从Qwen3-Coder-Next的推理轨迹进行知识蒸馏：
1. 收集170,356个多轮对话样本
2. 使用Qwen3-4B-Instruct-2507作为基座模型
3. 在4x NVIDIA H200 141GB上训练约25小时
4. 使用MS-SWIFT框架

### 4.2 模型规模

| 参数 | 规模 |
|------|-----|
| 模型参数量 | 4B |
| 基座模型 | Qwen3-4B-Instruct-2507 |

| 训练数据量 | 170,356样本 |
| 序列长度 | 16,384 tokens |
| 训练硬件 | 4x NVIDIA H200 141GB SXM5 |
| 训练时间 | ~25小时 |

## 5. 实验流程与结果

### 5.1 实验设置

#### 5.1.1 数据集
评估使用65个来自不同开源项目(scipy, fastapi, arrow, attrs, gevent, gunicorn等)的多轮对话样本，由Qwen3-Coder-Next生成标签。

### 5.2 主实验结果

#### 5.2.1 核心指标

| 指标 | 得分 |
|------|------|
| Tool Call Presence Alignment | 100% (65/65) |
| First Tool Type Match | 65.6% (40/61) |
| JSON Validity | 100% (76/76) |
| Argument Syntax Correctness | 100% (76/76) |

#### 5.2.2 JSON有效性对比

| 模型 | JSON有效 | 参数语法正确 |
|------|---------|------------|
| **LocoOperator-4B** | 76/76 (100%) | 76/76 (100%) |
| Qwen3-Coder-Next (teacher) | 89/89 (100%) | 78/89 (87.6%) |

### 5.3 局限性分析

#### 5.3.1 已知的局限性

1. **First-tool-type match为65.6%**: 模型有时选择与老师不同的工具(但不一定错误)
2. **工具调用数量偏少**: 相比teacher模型(89个)只生成76个工具调用
3. **评估规模有限**: 仅在65个样本上评估，需要更大规模评估
