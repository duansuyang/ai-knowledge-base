# superpowers - Agentic Skills 框架

## 1. 项目概述

### 1.1 项目名称
superpowers

### 1.2 一句话概述
一个完整的软件工程工作流框架，为编码 Agent 打造，让 Agent 在写代码前先理解需求。

### 1.3 详细描述
superpowers 是由 @obra (Jesse Vincent) 开发的一个 Agentic Skills 框架。它基于可组合的"技能"(skills) 构建，包含完整的软件开发方法论。Agent 不会直接跳入写代码，而是先理解需求、设计方案、制定计划，然后通过 Subagent 驱动的方式执行任务。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 强制性的技能驱动工作流
- **描述**: Agent 不会直接写代码，而是通过 skills 强制执行特定工作流
- **为什么重要**: 确保 Agent 按照最佳实践执行任务，避免"一上来就写代码"的常见问题
- **对比现有方法**: 传统 Agent 只是提供建议，而 superpowers 强制执行设计-计划-实现流程

#### 创新点 2: Subagent 驱动开发
- **描述**: 将大型任务分解为子任务，由 subagent 并行执行
- **为什么重要**: 让 Agent 能够自主工作数小时而不偏离计划
- **对比现有方法**: 单一 Agent 难以处理复杂多步骤任务

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|----------|
| 流程规范化 | 强制 TDD、YAGNI、DRY 实践 | 100% 强制执行 |
| 自动化 | Subagent 自主执行 | 可自主工作数小时 |
| 多平台支持 | Claude Code、Cursor、Codex、OpenCode | 4个主流平台 |

### 3.3 应用场景

1. **复杂软件项目开发**: 需要多人协作的大型项目
2. **代码审查流程**: 自动化代码审查和质量把控
3. **调试和问题解决**: 系统化的调试方法论
4. **团队知识传承**: 标准化的开发流程

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 基本工作流

```
1. Brainstorming (头脑风暴)
   → 激活条件: 发现要构建内容
   → 通过提问理解真正需求
   → 产出设计文档

2. Writing Plans (制定计划)
   → 激活条件: 设计已批准
   → 分解为 2-5 分钟的小任务
   → 每个任务有精确文件路径和验证步骤

3. Subagent-Driven Development (子代理驱动开发)
   → 激活条件: 有执行计划
   → 每个任务分配给 fresh subagent
   → 两阶段审查 (spec合规性 → 代码质量)

4. Test-Driven Development (测试驱动开发)
   → 激活条件: 实现过程中
   → RED-GREEN-REFACTOR 循环
   → 测试前的代码会被删除
```

### 7.2 综合评价

superpowers 是 Agent 工具领域的重要创新。它不只是提供建议，而是通过 skills 强制执行完整的软件开发流程。从需求理解到设计、从计划到执行、从测试到审查，每一步都有对应的 skill 强制执行。这代表了 AI 编程助手的下一阶段：从"建议者"变成"执行者"。

### 7.3 值得关注的点

1. **强制流程 vs 建议流程**: 区别于传统 Agent 的建议式交互
2. **Subagent 架构**: 支持长时间自主工作
3. **多平台支持**: 主流 AI 编码工具都已集成
