## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: SuperAgent Harness 架构
- **描述**: 不仅仅是一个框架，而是一个完整的 harness（马具/骨架），为 Agent 提供执行基础设施
- **为什么重要**: 传统 Agent 只是"有工具的聊天机器人"，DeerFlow 给 Agent 一台"自己的电脑"
- **对比现有方法**: 从 Deep Research 扩展为通用 SuperAgent，支持各种复杂任务

#### 创新点 2: 沙箱执行环境
- **描述**: 每个任务在隔离的 Docker 容器中运行，有完整的文件系统
- **为什么重要**: Agent 可以真正执行代码、读写文件，而不是"纸上谈兵"
- **对比现有方法**: 支持本地、Docker、Kubernetes 三种沙箱模式

#### 创新点 3: Sub-agent 并行执行
- **描述**: 主 Agent 可以动态生成 sub-agent，每个有独立上下文和工具
- **为什么重要**: 复杂任务可以并行探索不同角度，然后汇总为统一输出
- **对比现有方法**: 支持并行和串行两种 sub-agent 模式

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |

## 4. 技术细节

### 4.1 核心方法/架构

#### 4.1.1 系统架构

```
DeerFlow 2.0
├── Frontend (TypeScript)
│   └── 用户界面 (localhost:2026)
├── Backend (Python)
│   ├── LangGraph Agent
│   ├── Skills System
│   ├── Memory System
│   └── MCP Integration
├── Sandbox (Docker/K8s)
│   ├── 文件系统 (/mnt/user-data)
│   ├── 工作区 (/mnt/workspace)
│   └── 技能 (/mnt/skills)
└── LangChain/LangGraph
```

#### 4.1.2 核心功能

1. **Skills & Tools**
   - Skills 是结构化能力模块 (Markdown)
   - 内置: research, report-generation, slide-creation, web-page, image-generation
   - 按需加载，保持 context 精简

2. **Sub-Agents**
   - 复杂任务分解为子任务
   - Sub-agent 有独立隔离上下文
   - 支持并行执行和结果汇总

3. **Sandbox & File System**
   - 隔离的 Docker 容器
   - 完整文件系统访问
   - 可执行 bash 命令
   - 零污染，会话间完全隔离

4. **Context Engineering**
   - 独立 Sub-agent 上下文
   - 自动摘要和压缩
   - 长期记忆 (跨会话)

### 4.2 模型支持

支持任何 OpenAI 兼容 API 的 LLM，推荐:
- 长上下文 (100k+ tokens)
- 推理能力
- 多模态输入
- 强工具调用能力
