# MFLUX - MLX Image Generation

## 1. 项目概述

### 1.1 项目名称
MFLUX

### 1.2 一句话概述
在 Apple Silicon Mac 上本地运行最新的生成式图像模型。

### 1.3 详细描述
MFLUX 是 Huggingface Diffusers 和 Transformers 库中最先进的生成式图像模型的纯 MLX 移植版本。所有模型均使用 MLX 从头实现，仅使用 Huggingface Transformers 的 tokenizer。遵循 minimal and explicit 风格。

## 3. 项目优势

### 3.1 核心创新点

**创新点 1: 纯 MLX 实现**
- 描述: 所有模型使用 MLX 从头实现，不依赖 PyTorch
- 为什么重要: 充分发挥 Apple Silicon 的性能优势

**创新点 2: Minimal 风格**
- 描述: 保持代码简洁清晰，参考 karpathy 风格
- 为什么重要: 易于理解和修改

**创新点 3: 多模型支持**
- 描述: 支持 Flux, Qwen-image, Z-Image 等多种生成模型
- 为什么重要: 用户可以根据需求选择最合适的模型

### 3.2 技术优势

| 优势类别 | 具体优势 | 量化指标 |
|---------|---------|---------|
| 性能 | Apple Silicon 原生优化 | GPU 加速 |
| 内存效率 | MLX 内存优化 | 低显存占用 |
| 模型丰富 | 多个 SOTA 模型 | 5+ 模型 |

### 3.3 应用场景

1. **本地图像生成**: 无需云端，在 Mac 上本地生成图像
2. **快速原型**: 开发者快速测试图像生成效果
3. **隐私敏感场景**: 数据不需要离开本地设备

## 4. 技术细节

### 4.1 支持的模型

| 模型 | 描述 |
|------|-----|
| Flux | 最新的文本到图像模型 |
| Qwen-image | 阿里 Qwen 图像模型 |
| Z-Image | 高效图像生成模型 |
| SeedVR2 | 视频生成模型 |

### 4.2 安装方法

```sh
uv tool install --upgrade mflux
```

### 4.3 使用示例

```sh
mflux-generate-z-image-turbo \
  --prompt "A puffin standing on a cliff" \
  --width 1280 \
  --height 500 \
  --seed 42 \
  --steps 9 \
  -q 8
```
