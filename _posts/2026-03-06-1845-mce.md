# MCE - Model Context Engine

## 1. 项目概述

### 1.1 项目名称
MCE (Model Context Engine)

### 1.2 一句话概述
Token感知的透明代理，消除AI代理的上下文窗口膨胀问题。

MCE作为AI代理和MCP工具服务器之间的透明反向代理，拦截每个工具响应，评估其token成本，并应用3层压缩管道，在标准硬件上以接近零延迟运行。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 透明反向代理架构
- **描述**: MCE作为透明反向代理，位于AI代理和MCP服务器之间，无需修改现有代码
- **为什么重要**: 可以无缝集成到现有工作流中
- **对比现有方法**: 相比直接在应用层压缩，更加透明和无侵入

#### 创新点 2: 三层压缩管道
- **描述**: L1修剪(Pruner) + L2语义过滤(Semantic) + L3可选LLM总结(Synthesis)
- **为什么重要**: 分层压缩可以在保持关键信息的同时最大化压缩率
- **对比现有方法**: 相比单一压缩方法，效果更好

#### 创新点 3: 语义缓存
- **描述**: 零token即时响应重复请求
- **为什么重要**: 大幅减少重复请求的token消耗

## 4. 技术细节

### 4.1 核心方法

#### 4.1.1 三层压缩管道

1. **L1: Pruner (修剪层)**
   - 剥离HTML、base64 blob、空值、过多空白
   - CPU级别处理，速度极快

2. **L2: Semantic (语义层)**
   - 通过CPU友好的RAG提取相关块
   - 仅保留与任务相关的内容

3. **L3: Synthesis (合成层)**
   - 可选：路由到本地模型(Ollama)进行最终压缩
   - 使用LLM进行智能总结

#### 4.1.2 其他功能

- **语义缓存**: 零token即时响应重复请求
- **策略引擎**: 阻止破坏性命令(rm -rf, DROP TABLE等)
- **断路器**: 检测无限工具调用循环
- **实时仪表盘**: TUI显示token节省和缓存统计

### 7.2 综合评价

MCE是一个针对AI代理上下文窗口膨胀问题的创新解决方案。通过透明反向代理架构和三层压缩管道，可以节省高达90%的token使用。对于使用Claude Code、Cursor、Windsurf等AI代理的开发者来说，这是一个值得尝试的工具。

### 7.3 值得关注的点

1. 透明代理架构 - 无需修改现有代码
2. 三层压缩管道 - 分层压缩效果好
3. 语义缓存 - 重复请求零token消耗
4. 安全功能 - 内置破坏性命令阻止
