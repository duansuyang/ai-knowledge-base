# RPG Maker Agent

## 1. 项目概述

### 1.1 项目名称
RPG Maker Agent

### 1.2 一句话概述
给AI一篇文章，自动生成一款可以玩的RPG Maker MZ游戏

### 1.3 详细描述
该项目利用LLM将文章内容自动转换为可运行的RPG Maker MZ游戏。只需输入文章，AI会分析内容并生成游戏场景、事件和对话。支持分步生成、批注修改和一键导出。

## 3. 项目优势

### 3.1 核心创新点

#### 创新点 1: 文章转游戏管线
- **描述**: 将文章内容自动解析为游戏结构（场景、事件、对话）
- **为什么重要**: 降低了游戏创作门槛，非技术人员也能创作RPG游戏
- **对比现有方法**: 传统RPG制作需要手动设置大量参数，本项目自动化了这一过程

#### 创新点 2: 分步生成与审查
- **描述**: 6个AI管线步骤，每步可暂停检查和批注
- **为什么重要**: 确保生成质量，允许用户介入调整
- **对比现有方法**: 传统AI生成是一键式，无法精细控制

### 3.2 技术优势

| 优势类别 | 具体优势 | 说明 |
|---------|---------|-----|
| 易用性 | 只需输入文章 | 无需编程基础 |
| 可控性 | 分步生成+批注 | 精细控制输出 |
| 完整性 | 导出完整工程 | 可直接运行 |

### 3.3 应用场景

1. **内容创作**: 将小说、故事转换为互动游戏
2. **教育娱乐**: 让孩子学习的故事变成可玩游戏
3. **创意原型**: 快速搭建1-3个场景的Demo

## 4. 技术细节

### 4.1 核心方法

#### 4.1.1 AI管线流程
1. 新建项目 → 输入文章
2. 分步生成 → 6个AI管线步骤
3. 审查批注 → 可添加修改建议
4. AI修改 → 根据批注自动修订
5. 手动微调 → JSON编辑器
6. 生成工程 → 导出完整游戏

### 4.2 技术栈

| 类别 | 技术/工具 |
|------|----------|
| 框架 | TypeScript, Node.js |
| 前端 | Next.js |
| 游戏引擎 | RPG Maker MZ |
| AI | LLM API |

## 5. 实验与局限

### 5.1 当前状态
- 仍处于原型阶段
- 适合快速搭建1-3个场景的Demo
- 目前只有场景、事件和对话，战斗系统未实装

### 5.2 局限性

1. **场景数量**: 建议控制在1-3个场景
2. **战斗系统**: 暂未支持
3. **复杂逻辑**: 复杂事件可能需要手动调整
